Aula 11 - Métodos Especiais

A visibilidade define o nível de acesso aos atributos e métodos de uma classe.

Toda classe, independente de sua finalidade, possui dois métodos especiais que gerenciam o ciclo de vida do objeto: o construtor e o destrutor.

Métodos Especiais Fundamentais

Método Construtor

O método construtor é o responsável direto pela instanciação (criação) de objetos a partir de uma classe.

Tem como função alocar o objeto na memória (heap) e serve para prover valores iniciais (estado inicial) que o objeto deve possuir no momento de sua criação. Em linguagnes como Java e C#, deve obrigatoriamente ter o mesmo nome da classe e não possui tipo de retorno, omitindo inclusive a palavra void. É possível definir múltiplos construtores em uma classe através da sobrecarga, permitindo diferentes formas de inicialização.

Método Destrutor

O método Destrutor realiza a função inversa ao construtor, servindo para eliminar objetos da memórai quando não são mais necessários.

Durante a destruição o método é responsável por liberar recursos que o objeto possa ter alocado, como portas de comunicação ou arquivos abertos.

Observação: Em Java, a partir da versão 9, o uso do método finalize() (o destrutor padrão do Java) é desaconselhado (deprecated), pois erros em sua execução podem interferir no funcionamento do Garbage Collector. Recomenda-se o uso de alternativas como Cleaners ou a estrutura try-with-resources.

Métodos Auxiliares de Grande Utilidade (Herdados de Object)

Embora o foco estrutural recaia sobre contrututores e destrutores, todas as classes herdam automaticamente da classe base Object métodos essenciais para a manipulação de dados.

  • equals: Define os critérios para verificar se dois objetos são logicamente iguais, comparando seus estados internos em vez de apenas seus endereços de memória (o que o operador == faria).
  • hashCode: Gera uma representação numérica baseada no estado atual do objeto. Esse código é crucial para otimizar pesquisas em estruturas de dados que utilizam tabelas de dispersão (hash tables), como mapas e conjuntos
  • toString: Fornece uma representação padrão em texto do objeto. É uma alternativa mais encapsulada e amigável para exibir informações do que concatenar manualmente várias chamadas de métodos de acesso.

Métodos Acessores (get e set) e Encapsulamento

Os métodos acessores são a interface pública para manipulação de dados internos protegidos.

Eles são métodos públicos usados para consultar (get) ou alterar(set) o valor de atributos que, por boa prática devem ser definidos como privados para garantir a ocultação da informação.

O uso indiscriminado desses métodos pode ferir o encapsulamento, agindo como uma "quebra de blindagem" do estado interno. Permitir que o estado seja alterado livremente de fora via set pode levar a comportamentos adversos e é comparado a programar de forma estruturada dentro da Orientação a Objetos.
