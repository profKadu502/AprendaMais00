Aula 01 — A Natureza do Software

Esta aula apresenta os conceitos fundamentais sobre a natureza do software, suas características, tipos, o contexto histórico da crise do software e os princípios que fundamentam a Engenharia de Software.

Definindo Software

O software não se limita a programas executáveis. Ele é um conjunto integrado de:

  • Instruções que comandam o hardware
  • Estruturas de dados que organizam informações
  • Documentação que descreve uso e funcionamento
📘 Conceito: Software é um conjunto de instruções, dados e documentação que trabalham juntos para fornecer funções específicas.

Uma característica importante do software é sua natureza intangível. Ele não é físico e não sofre desgaste material. Porém, pode "envelhecer" devido a mudanças de requisitos, evolução tecnológica ou obsolescência.

Mesmo sem desgaste físico, o software pode apresentar falhas devido a erros de especificação, projeto ou codificação, o que exige o uso de metodologias que garantam qualidade e confiabilidade.

Características do Software

Diferente de produtos físicos, o software é desenvolvido e não fabricado. Ele é resultado de um processo intelectual.

  • Não existe linha de produção repetitiva
  • É altamente personalizado
  • Possui alta complexidade lógica
  • Não se deteriora fisicamente
  • Sofre obsolescência tecnológica

O desenvolvimento de software exige equilíbrio entre rigor técnico e criatividade.

Tipos de Software

  • Software de Sistema: sistemas operacionais, utilitários e drivers
  • Software de Aplicação: sistemas de gestão, editores, jogos
  • Software Científico e de Engenharia: cálculos e simulações
  • Software Embarcado: presente em dispositivos eletrônicos
  • Software de Linha de Produtos: famílias de sistemas semelhantes
  • Aplicações Web: sistemas acessados via navegador
  • Software de Inteligência Artificial: aprendizado de máquina e automação

Essa diversidade mostra que os princípios são comuns, mas o contexto varia conforme a aplicação.

A Crise do Software

Na década de 1960, projetos de software enfrentaram diversos problemas, caracterizando a chamada "crise do software".

  • Atrasos frequentes
  • Custos acima do previsto
  • Baixa confiabilidade
  • Dificuldade de manutenção

Essa crise evidenciou a necessidade de métodos mais organizados, levando ao surgimento da Engenharia de Software.

A Engenharia de Software surgiu para aplicar processos sistemáticos, padronização e qualidade no desenvolvimento.

O Processo de Software

O processo de software é um conjunto estruturado de atividades para desenvolver sistemas.

  • Especificação: definição dos requisitos
  • Projeto e Implementação: desenvolvimento da solução
  • Validação: testes e verificação
  • Evolução: manutenção e melhorias

Não existe um único modelo ideal. Modelos tradicionais são mais rígidos, enquanto métodos ágeis são mais flexíveis.

Princípios da Engenharia de Software

  • Compreender o problema antes de desenvolver
  • Planejar o processo
  • Manter simplicidade
  • Revisar constantemente
  • Garantir qualidade desde o início
  • Manter comunicação eficiente
  • Adaptar-se às mudanças

Prática da Engenharia de Software

A prática da Engenharia de Software baseia-se em três pilares:

  • Métodos: técnicas de análise, projeto, codificação e teste
  • Ferramentas: IDEs, controle de versão, ferramentas CASE
  • Procedimentos: padrões e organização do trabalho
📌 Importante: O objetivo da Engenharia de Software é produzir sistemas confiáveis, econômicos e alinhados às necessidades do usuário.
💡 Dica: O sucesso de um sistema depende tanto da técnica quanto da comunicação com o cliente.
