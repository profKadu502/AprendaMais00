Git e GitHub

Aprenda a versionar seus projetos e enviar para a nuvem.

📌 O que é Git e GitHub?

O Git é um sistema de controle de versão que permite acompanhar mudanças em arquivos ao longo do tempo. Já o GitHub é uma plataforma online onde você pode armazenar e compartilhar seus projetos que utilizam Git.

🚀 Criando um repositório

Primeiro, acesse o site do GitHub e crie um novo repositório. Defina um nome para o projeto e evite marcar opções automáticas como README ou .gitignore neste primeiro momento.

💻 Inicializando o projeto

No terminal, dentro da pasta do seu projeto, execute:

git init

Esse comando inicia o Git no seu projeto, permitindo o controle de versões.

📂 Adicionando arquivos

Para adicionar todos os arquivos do projeto ao controle de versão:

git add .

📝 Criando um commit

O commit salva uma versão do seu projeto. Sempre escreva uma mensagem clara descrevendo a alteração: 

git commit -m "mensagem"

🌐 Enviando para o GitHub

Após conectar seu projeto ao repositório remoto, envie os arquivos com: 

git push

🔁 Fluxo básico do dia a dia

No uso diário, você sempre seguirá este fluxo: 


git add .
git commit -m "mensagem"
git push

Esse processo garante que suas alterações sejam salvas localmente e enviadas para o GitHub.
